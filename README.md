# Compilation Techniques Project

Implementation of the MiniImp and MiniFun programming languages developed for the **Compilation Techniques** course.

---

# Overview

This project implements a complete compiler and interpreter infrastructure for two educational programming languages:

- **MiniImp** – an imperative programming language
- **MiniFun** – a functional programming language

The implementation was developed incrementally through multiple project fragments following the topics presented during the course. It covers the complete compilation pipeline, including parsing, semantic evaluation, type checking, program analysis, optimization, and LLVM code generation.

---

# Implemented Fragments

- **Fragment 1** – MiniImp Interpreter
- **Fragment 2** – MiniFun Interpreter
- **Fragment 3** – MiniFun Type Checker
- **Fragment 5** – Control Flow Graph (CFG)
- **Fragment 6** – Dataflow Analysis
- **Fragment 7** – Compiler Optimizations
- **Fragment 8** – LLVM Backend


Build the project:

```bash

```

Build the optimized release version:

```bash
cargo build --release
```

# Running the Project

Execute the project:

```bash

```

Execute the optimized release version:

```bash
cargo run --release
```

The main program demonstrates the implemented compiler fragments, including interpretation, type checking, control-flow graph construction, dataflow analyses, compiler optimizations, and LLVM code generation.

---

# LLVM Compilation Pipeline

Generate the LLVM IR:

```bash
cargo run
```

Apply LLVM's SSA optimization:

```bash
opt -passes="mem2reg" program.ll -S -o program_opt.ll
```

Generate the object file:

```bash
llc -filetype=obj program_opt.ll -o program.o
```

Link the executable:

```bash
clang wrapper.c program.o -o program
```

Run the generated executable:

```bash
./program
```

---

# Generated Files

Running the LLVM backend produces the following files:

| File | Description |
|------|-------------|
| `program.ll` | Generated LLVM Intermediate Representation |
| `program_opt.ll` | LLVM IR after the `mem2reg` optimization |
| `program.o` | Object file generated by LLVM |
| `program` | Final executable |
